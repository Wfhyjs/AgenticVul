# AgenticVul 环境配置与运行指南

本项目分为后端（Python/Flask/PyTorch）和前端（Node.js/Vue 3）。请按照以下步骤配置依赖并运行项目。

## 1. 环境准备

确保您的操作系统（Windows/Linux/MacOS）已安装以下基础软件：
- [Miniconda 或 Anaconda](https://docs.conda.io/en/latest/miniconda.html)（推荐用于管理 Python 环境）
- [Node.js](https://nodejs.org/) (建议版本 v16 或 v18 以上，用于前端运行)
- Git

---

## 2. 后端配置 (3s_back)

后端采用 Python 构建，涉及大模型（LLM）驱动的检测和 Flask 原生接口。

### 2.1 创建并激活 Python 环境

打开终端（如 Anaconda Prompt 或 PowerShell），并在项目根目录 `AGenticVul` 下执行：

```bash
# 创建一个名为 agenticvul 的 python 3.10 环境
conda create -n agenticvul python=3.10 -y

# 激活环境
conda activate agenticvul
```

### 2.2 安装核心依赖库

进入后端目录并安装所需的 Python 库。根据之前安装记录，主要必需库如下：

```bash
cd 3s_back

# 安装基础 Web 框架与工具
pip install flask flask-sqlalchemy flask-cors flask-bcrypt pyjwt

# 安装深度学习与 AI 相关库
pip install torch torchvision torchaudio  # 建议根据CUDA版本去 PyTorch 官网获取对应安装命令
pip install transformers scipy tiktoken tree_sitter

# 安装大模型 API 调用所需库
pip install openai ell-ai sqlmodel "ell-ai[sqlite]"
```

---

## 3. 深度学习模型准备

由于代码审计依赖于底层的 Hugging Face 预训练模型（如 CodeBERT）以及作者微调的权重向量（.pth），您需要补全模型文件。

### 3.1 下载 Hugging Face 基础配置与 Tokenizer

在项目根目录下，我们已经准备了自动下载公共基础配置（不含G级大权重，仅含词表、配置等）的脚本：

```bash
# 回到根目录
cd ../

# 设置国内 Hugging Face 镜像加速并运行脚本
# Windows (PowerShell):
$env:HF_ENDPOINT="https://hf-mirror.com"
python download_base_models.py

# Linux/Mac:
HF_ENDPOINT=https://hf-mirror.com python download_base_models.py
```
*这会自动将 GraphCodeBERT、CodeBERT 等配置文件下载到 `3s_back/user_sys/DetectionModels/BaseModel/` 中。*

### 3.2 手动下载微调权重（.pth）

原始作者训练好的 `model.pth` 检测权重由于体积较大且为私有发布，不在 Hugging Face 上。
- **获取方式**：请前往原项目的 GitHub 仓库 `README.md` 中寻找百度网盘/Google Drive 的下载链接。
- **放置位置**：下载后，将 `.pth` 放入对应算法文件夹中（例如 `3s_back/user_sys/DetectionModels/DFEVD/model.pth` 等）。

*(注：系统目前具备 Mock 兜底机制，如果找不到真实 `.pth` 模型，仍会正常运行并返回随机的检测结果供 UI 展示。)*

## 4. 前端配置 (3s_front)

前端采用 Vue 3 和 Vite 构建。

打开一个**新的终端窗口**，进入项目根目录：

```bash
# 进入前端目录
cd 3s_front

# 安装 Node.js 依赖包
npm install
```

---

## 5. 本地运行项目

必须同时运行后端 API 服务和前端界面服务，项目才能正常工作。

### 5.1 启动后端
在已激活 `agenticvul` 的终端中：
```bash
# 确保在 3s_back 目录下
cd 3s_back
python app.py
```
启动成功后，后端接口将运行在 `http://127.0.0.0:5000`。

### 5.2 启动前端
在另外一个终端中：
```bash
# 确保在 3s_front 目录下
cd 3s_front
npm run dev
```
启动成功后，会自动显示本地前端访问地址（通常为 `http://localhost:5173` 或类似地址）。在浏览器中打开该地址即可使用 AgenticVul 系统！
